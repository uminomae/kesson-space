kesson-space design system

Design tokens, structural patterns, and component specifications used across kesson-space. Other repositories can reference this page for consistent visual language.

1. Structure Patterns

Section Pattern: Narration + Heading + Container

Each content section follows a consistent pattern: optional narration, a section heading, and a content container with panels or card grids.

Language Switching

Bilingual content uses data-ja and data-en attributes on leaf elements only. The page-language.js module reads these attributes and swaps textContent on language change. Only place these attributes on the innermost text-containing element.

<!-- Correct: data-* on the leaf text element --> <p class="room-narration-text" data-ja="欠けているものが、何かを指し示す。" data-en="What is missing points to something."> What is missing points to something. </p> <!-- Incorrect: data-* on a wrapper --> <div data-ja="..." data-en="..."> <p>text</p> </div>
