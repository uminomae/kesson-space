# 表現一覧（Expression Index）

**目的**: 理論の核心を読者に届ける「一文表現」を管理する。

**設計意図**:
- schema/（構造）で定義された概念を、読者が受け取れる言葉に変換
- 各表現は群盲象チェック（3領域以上）を通過したものを adopted とする
- 読者別バリエーション（研究者・経営者・一般 等）を各ファイルに格納
- concepts/（概念ノート）が「なぜそう言えるか」、expression/が「どう伝えるか」

## テンプレート（必須セクション）

各表現ファイルは以下を含む：

1. **frontmatter**（id, status, text, text_en, abstraction, related_definitions, related_modules）
2. **群盲象チェック**（3領域以上）
3. **バリエーション**（読者別表現）
4. **使用履歴**（どの出力で使用されたか）
5. **検討経緯**
