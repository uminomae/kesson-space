# WORKFLOW - セッション管理

## セッション開始時（Claude向け）

```
1. docs/CURRENT.md を読む
2. 前回の「次セッションのタスク」を確認
3. 未決定事項を把握
4. 必要に応じて docs/CONCEPT.md, docs/ARCHITECTURE.md を参照
```

### 開始時の確認項目

- [ ] 前回どこまで進んだか
- [ ] 今回何をするか
- [ ] ブロッカーはあるか

---

## セッション終了時（Claude向け）

```
1. docs/CURRENT.md を更新
   - 完了タスクにチェック
   - 新しい未決定事項を追記
   - 次セッションのタスクを設定
   - 技術的メモを更新
2. 必要に応じて他のdocsを更新
3. GitHub にプッシュ（またはMCPで直接更新）
```

### 終了時の記録項目

- [ ] 今回何を完了したか
- [ ] 何が未完了か
- [ ] 新しく発生した問題・疑問
- [ ] 次回最初にやるべきこと

---

## CURRENT.md のテンプレート

```markdown
# CURRENT - 進捗・引き継ぎ

**最終更新**: YYYY-MM-DD
**セッション**: #N

---

## 現在の状態

### 完了
- [x] タスク

### 進行中
- [ ] タスク

### 未着手
- [ ] タスク

---

## 未決定事項

| 項目 | 選択肢 | メモ |
|------|--------|------|
| ... | ... | ... |

---

## 次セッションのタスク

1. ...
2. ...

---

## 技術的メモ

- ...

---

## 参照リンク

- [CONCEPT.md](./CONCEPT.md)
- [ARCHITECTURE.md](./ARCHITECTURE.md)
```

---

## セッション番号の採番

- 新しいセッションごとに `#N` をインクリメント
- 日付が変わったら新セッション
- 長時間空いたら新セッション
