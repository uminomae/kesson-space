## index.html

ランディング。Three.js Raymarch SDF 背景 + Bootstrap offcanvas/modal + セクション列（HERO → GUIDES → RESEARCH → ARTICLES → FOOTER）。

### レイアウト
- **Three.js canvas + overlay content の z-index 分離**（canvas-container z-index:1 / section-content-wrap z-index:20 / topbar 上層）: 背景は常時描画、本文は透過パネル越しに読ませる。KEEP — kesson-space の視覚的アイデンティティそのもの。
- **hero-spacer / articles-spacer による強制スクロール余白**: カメラ下降演出とセクション出現のタイミングを「スクロール距離」で制御。JSで位置追従するより単純で壊れにくい。KEEP
- **`section > .section-content-wrap > .container`** 三層ラップ: 背景・余白・内容幅を層で分離。Bootstrap container と統合しても機能するが、z-index 責務を明示できるので現行維持。SYSTEMIZE（共通 component 化）
- **`.ks-card-grid` / `.ks-card-grid--2col`**: Bootstrap row-cols に加えた独自 grid。`row row-cols-1 row-cols-md-2 g-3` の命名を短縮したエイリアス。SYSTEMIZE

### アニメーション
- **スクロール連動カメラ下降（scroll-coordinator.js + hero-spacer）**: ヒーローで「潜水」する感覚を作る。kesson-driven thinking の「欠損の底に降りる」メタファー。KEEP — 意図が概念と直接対応
- **`room-narration-text` の静的配置**（アニメなし、ただ現れる）: 幕間の沈黙を作る。アニメで動かすと軽薄になる。KEEP（アニメ不採用が意図）
- **カード hover: `translateY(-2px)` + soft shadow / 0.2s ease**: 触覚的フィードバックの最小量。強く持ち上げないのが kesson らしさ。KEEP
- **`text-shadow: 0 0 24px rgba(accent, 0.08)`**（room-narration-text）: 明朝体に微かな光彩。読みにくくならない強度に留めている。KEEP。ただしスクロール性能を落とす可能性があるため負荷監視は必要（backdrop-filter と合わせた検証）。

### 配色
- **`--color-bg-body: #050508`**（ほぼ黒、わずかに青）: 純黒を避け、accent 青と調和させる。KEEP
- **`--color-accent: 100, 150, 255`**（薄い青）: Raymarch SDF の光と同系色。SYSTEMIZE（共通 accent として pd 集約候補）
- **`rgba()` を通したトーン階調**（0.1 / 0.15 / 0.3 / 0.5 / 0.6 / 0.7 / 0.85 / 0.9）: 明度差でヒエラルキーを作る。灰色諧調ではなく accent を透過で薄める技法。KEEP — 暗背景での可読性を保つ
- **`--kesson-card-bg: rgba(20, 25, 40, 0.9)`**（半透明濃紺）: 背景 Three.js を透かせる。canvas 描画がカードに干渉しない強度。KEEP
- **`--color-highlight / --color-sub-text / --color-heading` の3階調**: 白・薄青白・薄青の三段。タイトル/本文/補助を明示的に分離。SYSTEMIZE（共通 token）

### タイポグラフィ
- **和文: Noto Serif JP 300 / 400**: 明朝の細字で「思索的な静けさ」。見出しにも本文にも同一 family、weight だけ振る。KEEP — kesson-driven thinking の理論性と親和
- **欧文 UI: SF Mono / Fira Code / Consolas** (section-heading, eyebrow, labels): 技術実直さ。明朝（概念）と等幅（実装）を対比させる二本柱。KEEP
- **`--kesson-font-serif-ui: 'Georgia'`**: カードタイトルの欧文用。Noto Serif JP とのフォールバック。KEEP
- **letter-spacing トークン4段（tight/normal/wide/heading: 0.03 / 0.06 / 0.1 / 0.15em）**: 見出しは 0.15em で「空気」を多く取る。SYSTEMIZE（共通 token）
- **`clamp(0.95rem, 2.8vmin, 1.35rem)`**（room-narration-text）: ビューポート依存の可変サイズ。vmin 基準でデバイス横断。KEEP

### 触覚的要素
- **`--kesson-radius-sm/md/lg: 2 / 3 / 4px`**（ほぼ直角）: ラウンドを最小限に留める。明朝 + mono の硬質さと整合。KEEP — 丸さを取るとブランド崩れる
- **`--kesson-card-shadow-soft: 0 4px 12px rgba(accent, 0.15)`**: 黒影ではなく accent 色影。光源が画面奥から来る錯覚。KEEP
- **backdrop-filter: blur(24px)**（viewer glass）: Modal で使用。Three.js 背景を残したまま本文を読ませる。KEEP（ただし performance 負荷は既知、現行維持）
- **glass box-shadow 三層**（0 0 60px + 0 0 120px accent + inset）: viewer の「浮遊感」。手作りの妙味。KEEP

### インタラクション
- **scroll-coordinator（scrollRestoration='manual' 先行注入）**: main.js の重い import より前に inline module で走らせる。Issue #4 の pivot 。KEEP — 強い制約ゆえの判断
- **Bootstrap offcanvas 右スライド（devlog / articles）**: ページ遷移なしの副次コンテンツ閲覧。Bootstrap の完成度に乗る。KEEP
- **`e2e-auto-run.js`** dev テストフック: 本番画面でも読み込まれるが、localhost 判定等で動作制御されている想定。要確認。
- **`section-heading-link`（アンカー to self）**: 各セクションが URL で引用単位になる。思想としてこのサイトの「参照可能性」を支える。KEEP
- **文字サイズ調整（A- / ↺ / A+）**: topbar に実装。読者の読書環境を尊重する明朝サイトらしさ。KEEP
- **言語切替（data-ja / data-en を leaf only）**: page-language.js が textContent を swap。wrapper に置かない規約。KEEP（規約として明文化済み、design-system.html に記載）

## サマリ

### 定量

- **KEEP 要素数**: 約 38（全セクション合計）
- **SYSTEMIZE 要素数**: 約 11（観点B で token / component 化候補）
- **DROP 要素数**: 2（devlog.html の disabled コード長大、未使用 dev-panel token の一部候補）

### 全 repo 共通化できそうな要素（観点B で pd 集約優先）

1. **color token セット**: `--color-accent / --color-sub-text / --color-highlight / --color-heading / --color-bg-body / --color-link`（6 core + 拡張）
2. **letter-spacing 4段トークン**（tight / normal / wide / heading）
3. **radius 3段トークン**（sm/md/lg）
4. **transition 2段トークン**（standard / snappy）
5. **明朝 + mono の二本柱フォント規約**（Noto Serif JP + SF Mono）
6. **section-heading（mono uppercase 0.15em）+ section-heading-link（self anchor）**
7. **kesson-card（h-100 / translateY hover / soft shadow）**
8. **room-narration + room-narration-text（幕間ナレーション）**
9. **card grid 2col/3col（Bootstrap row-cols ベース）**
10. **viewer glass（bg/border/shadow/backdrop-filter）**
11. **`section > .section-content-wrap > .container` 三層ラップ（z-index 責務分離）**

### 判断の非自明点

- **明朝 + mono の二本柱は kesson-space の署名**: この組み合わせを他 3 repo が同じ比率で採用するか、repo ごとの比率を設計するかは design system 抽出時の論点。kesson: 明朝主 / mono 補。pd: 明朝主 / quartz は独自。as / cs: 未確認（B フェーズで判断）。
- **token 名のプレフィックス `--kesson-*`**: 現行は repo 名が混入している。pd 集約時は `--pd-*` に統一するか、汎用名（`--ds-*`）か、repo プレフィックス維持か、が C フェーズの決定事項。
- **backdrop-filter + text-shadow の二段使い**: 視覚効果は強いがスクロール性能を落とす既知パターン（プロジェクト memory: 「スクロール jank は CSS 起因」）。token 化時に「重量クラス」として分類すると、低スペック端末向けに切り替え可能。

### 気づき（観点B で先に決めたほうが良いこと）

- **`--kesson-card-title / --kesson-card-text / --kesson-section-heading` の font-size が tokens.css と design-system.html で微妙に異なる**: tokens.css は `--kesson-section-heading: 0.88rem`、design-system.html の live 記述は `0.80rem`。B フェーズで正本を決める必要あり。
- **`--kesson-viewer-glass-*` と `--kesson-md-*` の責務重複**: glass surface 用の色と markdown 表現用の色が別系統で増殖。統合すると token 数半減できる可能性。
- **`design-system.html` を**単一ソース**として扱うなら、live example を各 repo が引用できる形（iframe or web component）が必要**: 現状は HTML を手でコピーする形。観点C（Claude Design 連携）の仕様書フォーマット決定と並走すべき。
