# GUIDES 構成憲章

> kesson-space の GUIDES セクションが何を伝え、どう構成するかの設計原則。
> 2026-03-24 セッションで確定。

---

## 1. 出発点と根本の問い

### 1.1 表の出発点

**「無意味さは無意味なのか？」**

これが全ての始まりである。D1-D4 は「4つの仮説」ではない。ただの出発点であり、問いの入口にすぎない。

### 1.2 裏にある根本的な問い

**「信頼とは何か？」**

欠損駆動思考は、信頼を獲得するための思考・構えではないか。
無意味さにすら意味を見出す態度 — それは信頼の基盤を形成する。

### 1.3 信頼の暫定的な輪郭（未確定・探索中）

信頼の定義はまだ曖昧だが、2つの仮説的な手がかりがある:

1. **変わらないことが鍵**: 信頼は一貫性・恒常性に根ざすのではないか。D3（抱持）の「反射的に処理せず保持する」機能 — 揺れずに留まること — が信頼の構造的条件かもしれない。

2. **価値の二軸**: この世の価値のエネルギーは二軸で形成されているように見える:
   - **生存軸（生存）→ お金**（物質的・生存のための価値）
   - **信頼軸（間主観性）→ 信頼**（関係的・間主観的な価値）

   これは D4 の 生存-信頼 評価軸と対応する。欠損駆動思考が扱う「情動の構成」は、まさにこの二軸上での評価プロセスである。

> **注意**: §1.2-1.3 は探索段階の記録であり、確定した論ではない。GUIDES に直接反映する前に、kesson-driven-thinking の理論的検証を経る必要がある。

## 2. 論の構造

### 2.1 D1+D2 はセット

| 定義 | 内容 | 性質 |
|---|---|---|
| **D1** | 棄却される誤差を、問いとして拾う態度 | 態度（メタ） |
| **D2** | 予想と現実の誤差を、意識が「欠け」として捉えた主観的経験 | 経験 |

D1（態度）は D2（経験）と切り離せない。「拾う」という態度は、「欠けを感じる」という経験があって初めて成立する。

### 2.2 これは創造的な思考・構えである

欠損駆動思考は単なる認知モデルではなく、**創造的な構え**と呼べる。
したがって「創造とは何か？」を調査した creation-space の知見を外部参照する。

### 2.3 creation-space の知見（外部参照）

creation-space は 30 の学術領域を横断し、創造プロセスの構造類似パターンを調査した。
その結果、5つの横断テーマが抽出された:

| テーマ | 欠損駆動思考との接点 |
|---|---|
| **縁の類型学**（28/30領域） | 縁は「線」ではなく「帯域」。方向が選択される場 → D4（生存-信頼評価）の構造的裏付け |
| **再循環メカニズム**（19/30領域） | 束は終点ではなく次の場の種。欠損は一回性ではなく循環する |
| **場の多層性**（15/30領域） | 場は空虚ではなく「条件付き潜在性」→ D1 の態度が場の質を決める |
| **盲点と反例**（13+11/30領域） | モデル自身の欠損（解体を記述できない）が生産的 → 欠損駆動思考の自己言及 |
| **閾値構造**（11/30領域） | 縁→渦の遷移は閾値的 → D3（抱持）は閾値管理として読める |

**参照方法**: GUIDES から creation-space のレポートに外部リンクする。丸ごと取り込まない。

### 2.4 D2+D4 は awareness-space に詳細がある

| 定義 | 詳細の所在 |
|---|---|
| **D2** 欠損（主観的経験） | awareness-space |
| **D4** 情動の構成（生存-信頼評価） | awareness-space |

**参照方法**: GUIDES から awareness-space に外部リンクする。

## 3. GUIDES の構成方針

### 3.1 現状（3ガイド × 外部 pjdhiro）

| ガイド | 対象 | 外部参照先 |
|---|---|---|
| kesson-general | 一般向け | pjdhiro/assets/kesson/guides/ |
| kesson-designer | 設計者向け | 同上 |
| kesson-academic | 専門家向け | 同上 |

### 3.2 拡張方針

GUIDES に creation-space と awareness-space の知見を**外部参照**として組み込む:

```
GUIDES
├── 出発点: 「無意味さは無意味なのか？」
│
├── D1+D2: 態度と経験（セット）
│   └── 詳細 → awareness-space（D2, D4）
│
├── これは創造的な構えである
│   └── 根拠 → creation-space（30領域横断調査）
│       ├── 5段階モデル（場→波→縁→渦→束）
│       └── 5つの横断テーマ
│
├── 既存3ガイド（一般/設計者/専門家）
│   └── 外部参照 → pjdhiro
│
└── 帰結: 欠損は欠落ではなく、創造の駆動力である
```

### 3.3 外部参照の原則

- kesson-space 内にコンテンツを複製しない
- 各リポジトリの正本へリンクする
- リンク切れ時のフォールバックは不要（GitHub Pages 上の安定URL）

| 参照先 | URL パターン |
|---|---|
| creation-space | `https://uminomae.github.io/creation-space/` |
| awareness-space | `https://uminomae.github.io/awareness-space/` |
| pjdhiro（ガイド） | `https://uminomae.github.io/pjdhiro/assets/kesson/` |

### 3.4 解説書ガイドの文体基準

3ガイド（kesson-general / kesson-designer / kesson-academic）のコンテンツは、`transform/rules/slide-content-rules.md` §3 のプレゼンテーション文体基準に従う。

| 基準 | 内容 |
|---|---|
| 文体 | 「です・ます」調 |
| 語彙 | 中学生が読んで意味がわかる日本語のみ |
| 文の長さ | 1文40字以内を目安 |
| 前提知識 | ゼロ |
| 禁止 | 論文調、学術用語、内部用語、リポジトリ名、リンク/URL、LLM的創作、過大主張 |

#### 用語統一

| 禁止用語 | 正式用語 |
|---|---|
| Withhold | 抱持 |
| Container | 支えとなる関係性 |
| 生存軸 / 信頼軸 | 生存の軸 / 信頼の軸 |
| D1-D4 | 概念を日常語で説明する |

#### kesson-academic の例外

kesson-academic は専門家向けであるため、以下の例外を認める:

- 学術用語の使用は、初出時に平易な説明を併記すれば許容する
- 「である」調は許容する（学術文書の慣例に従う）
- 先行研究の著者名・文献名の引用は許容する

ただし、以下は例外なく適用する:

- 内部用語（D1-D4、生存軸/信頼軸 等）は初出時に説明を付ける
- リポジトリ名は使用しない
- 過大主張・LLM的創作は禁止
- 「Withhold」は「抱持」に統一

## 4. コンテンツ制作ワークフロー

### 4.1 パイプライン全体像

```
構想・探索（本セクション）
    ↓
ドラフト（Markdown, layout ヒント付き）
    ↓
生成（generate-rich-slides.py）
    ↓
自己完結 HTML（16:9, CSS/JS インライン）
    ↓
表示（openRichSlideViewer → iframe）
```

### 4.2 構想から生成までの軌跡

| ステップ | 成果物 | 参照先 |
|---|---|---|
| **構想**: 論の構造を設計 | 本憲章 §1-3 | `docs/CHARTER-guides.md` |
| **探索**: 理論的裏付けを調査 | 探索メモ | `docs/exploration-trust-hypothesis.md` (#147) |
| **ドラフト**: スライド原稿を執筆 | presentation MD | `content/guides/meta-overview.md` |
| **生成ルール**: デザインスペック | HTML シェル + CSS | `docs/rich-slides-design-spec.md` |
| **生成**: Python スクリプト | 自己完結 HTML | `scripts/generate-rich-slides.py` |
| **スキル**: LLM 向けスライド生成手順 | SKILL.md | `transform/rich-slides/SKILL.md` |

### 4.3 ドラフト作成の原則

- 憲章 §1-3 の論の構造に従う（問い → D1+D2 → 創造的構え → 外部参照 → 帰結）
- `---` で区切った Markdown がそのまま1スライド
- `<!-- layout: TYPE -->` で配置ヒントを付与（title, data, split, quote, conclusion）
- 外部参照は URL を記載（コンテンツ複製禁止）
- 探索メモ（#147 等）の確定事項のみドラフトに反映、未確定は含めない

### 4.4 生成コマンド

```bash
python3 scripts/generate-rich-slides.py content/guides/meta-overview.md
# → content/guides/meta-overview.html (自己完結 HTML)
```

## 5. 禁止事項

- D1-D4 を「4つの仮説」として提示すること（出発点であって仮説ではない）
- creation-space の調査結果を kesson-space 内に丸コピーすること
- awareness-space の D2/D4 詳細を kesson-space 内に複製すること
- 論の帰結を先に述べて出発点を省略すること（問いから始める）
