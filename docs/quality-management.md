# 品質管理基準

kesson-space プロジェクトの品質チェックポイント。

## 品質チェック（6項目）

### 1. config.js SSoT 整合性

- 全てのパラメータが `src/config/` に定義されているか
- マジックナンバーが散在していないか
- dev-ui のデフォルト値が config パラメータを参照しているか

### 2. シェーダー/JS 分離

- GLSL ロジックがJS側に漏れていないか
- uniform 名が config と一致しているか
- パフォーマンス: per-frame `new` アロケーションがないか

### 3. テスト網羅

- `node tests/config-consistency.test.js` がパスするか
- E2E smoke (TC-01, 02, 04) に問題がないか
- シェーダー変更時: before/after のスクリーンショット推奨

### 4. i18n 整合

- 新規文字列は ja/en 両方に追加されているか
- `sessions.json` のエントリに `title_en`, `summary_en` があるか

### 5. OPS テスト（運用整合性）

| # | チェック項目 |
|---|---|
| OPS-1 | この変更は手動作業を増やさないか？ |
| OPS-2 | docs/README.md と整合しているか？ |
| OPS-3 | 使用箇所から参照可能か？ |
| OPS-4 | 情報の重複を生んでいないか？ |
| OPS-5 | git 履歴でリカバリ可能か？ |

### 6. 目視確認（pjdhiro 専権）

- 60fps 安定（中程度のラップトップ基準）
- 描画の不具合なし
- 忍的美学（控えめ・抑制的）に沿っているか
- CONCEPT.md との整合

## コミット前チェック

1. `node tests/config-consistency.test.js` パス
2. ローカルサーバーで目視確認（`./serve.sh`）
3. コンソールエラーなし
4. 変更箇所に `// CHANGED(YYYY-MM-DD)` コメント（1ヶ月後に削除）

## レビュー基準

| 基準 | 内容 |
|---|---|
| 正確性 | config.js の値がシーンに正しく反映されるか |
| 完全性 | 変更の影響範囲が全てカバーされているか |
| パフォーマンス | FPS 低下がないか、メモリリークがないか |
| 再現性 | 同じパラメータで同じ結果が得られるか |
