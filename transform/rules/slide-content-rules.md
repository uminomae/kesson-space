## 1. 語彙ルール

### 1.1 内部記号の禁止

以下の記号・用語をスライド内で使ってはならない:

| 禁止 | 代替表現 |
|---|---|
| D1, D2, D3, D4 | 概念を日常語で説明する（下記 §1.2 参照） |
| M1, M2, M3, M4 | 「意識の4層」「創造の5段階」等の平易な表現 |
| F軸 / O軸 | 「生存の軸」「信頼の軸」 |
| L0, L1, L2, L3 | 層番号を使わず機能で説明する |
| kesson-driven-thinking | 「欠損駆動思考」（日本語）または概念説明で代替 |

### 1.2 概念の言い換え辞書
<!-- CHANGED(2026-03-25) §3 文体基準に整合: 論文調・感情的修辞を排除し、中学生が読める表現に統一 -->

| 内部概念 | 読者向けの表現 |
|---|---|
| D1（態度） | 違和感を手放さない構え / 「おかしい」と感じたことを大切にする姿勢 |
| D2（欠損） | 予想と現実のズレを「何かが足りない」と感じること / 引っかかり |
| D3（抱持） | すぐに答えを出さず、問いとして持ち続けること |
| D4（情動構成） | 生存に関わるか、人との関係に関わるかで、感じ方の方向が分かれること |
| Container | 問いを持ち続けることを支えてくれる関係性 |

### 1.3 禁止フレーミング
<!-- CHANGED(2026-03-25) §3 文体基準に整合: 内部参照「憲章 §1 違反」を §3.2 準拠に変更 -->

| 禁止 | 理由 | 代替 |
|---|---|---|
| 「5ステップで〜」 | 自己啓発調 | 「5段階の構造」 |
| 「あなたの創造性を高める」 | 処方的 | 「今どこにいるか」 |
| 「本モデルが既存理論を統合する」 | 過大主張 | 「異なるラベルが同じ構造を指しているかを見る」 |
| 結論から始める | §3.2 禁止事項 | 問いから始める |

## 2. コンテンツ構造

### 2.1 メタスライドの3部構成
<!-- CHANGED(2026-03-25) §3 文体基準に整合: リポジトリ名を排除 -->

```
【Part 1: 問い】
  「無意味さは無意味なのか？」
  → 違和感を捨てずに問いに変える態度がある

【Part 2: 創造とは何か】
  30の学術領域が見つけた構造:
  → 創造は「欠けているもの」によって駆動される
  → 終わりは次の始まりの種になる（循環）
  → 境界は線ではなく、方向が選ばれる帯域

【Part 3: 意識とは何か】
  身体 → ズレの検出 → 生存か信頼かの評価 → 問いとして保持
  → 意識は反応ではなく、問いを保持する能力

【帰結】
  欠損は欠落ではない。創造の駆動力である。
```

### 2.2 1スライド1メッセージ

- 各スライドは1つのメッセージだけを伝える
- 補足は箇条書き3項目以内
- 表は5行以内

### 2.3 外部参照の書き方
<!-- CHANGED(2026-03-25) §3 文体基準に整合: リポジトリ名の使用を禁止に変更 -->

- スライド内にリンク（URL）を埋め込むことは禁止
- リポジトリ名（creation-space、awareness-space 等）の使用は禁止。内容だけを説明する
- コンテンツの複製禁止

## 3. 文体基準

### 3.1 基本トーン: プレゼンテーション資料

本スライドは公開向けプレゼンテーション資料である。Apple・Google の製品紹介ページ、企業の IR 説明資料、TED トークのスライドと同等の文体基準を適用する。

| 基準 | 内容 |
|---|---|
| 文体 | 「です・ます」調。丁寧で平易な標準語 |
| 語彙 | 中学生が読んで意味がわかる日本語のみ使用する |
| 文の長さ | 1文40字以内を目安とする。短く、リズムよく |
| 構成 | 1スライド1メッセージ。補足は箇条書き3項目以内 |
| 前提知識 | ゼロ。この概念を初めて知る人が対象 |
| 視点 | 読者が「自分に関係ある」と感じられる具体性を持たせる |
| 情報密度 | 余白を活かす。詰め込まない。表は3行以内を推奨 |

### 3.2 禁止する文体・表現

| 分類 | 具体例 | 代替 |
|---|---|---|
| 論文調 | 「〜と呼ぶ」「〜である」「〜に他ならない」 | 「〜といいます」「〜です」 |
| 学術用語 | エポケー、vmPFC、現象学的、計算論的 | 意味を日本語で説明する |
| 内部用語 | creation-space、container-mapping、D1-D4 | 使用禁止。内容だけを説明する |
| 口語・会話体 | 「たぶん」「なんとなく」「だが」 | 丁寧な標準語 |
| 感情的修辞 | 「ざわつき」「もやもや」 | 具体的に説明する |
| 自己啓発調 | 「あなたの創造性を高める」 | 使用禁止 |
| 過大主張 | 「統合した」「証明した」 | 使用禁止 |
| LLM的創作 | 根拠のない心理描写・体験記述 | 使用禁止 |
| リンク・URL | スライド内のハイパーリンク全般 | 使用禁止 |
| リポジトリ名 | creation-space、awareness-space 等 | 使用禁止。プロジェクト内部の名称は外部向け文書に出さない |

### 3.3 用語統一

| 禁止用語 | 正式用語 | 理由 |
|---|---|---|
| Withhold | 抱持 | 全文書で統一 |

## 4. 論拠の要件

### 4.1 主張と論拠の対応

すべての主張は以下のいずれかに分類し、論拠を明示する:

| 分類 | 記法 | 説明 |
|---|---|---|
| [P] | 確立された事実 | 学術領域で広く認められた知見 |
| [M] | 構造的類似 | 異なる領域間の構造的対応の指摘 |
| [S] | 検証可能な仮説 | 今後の検証が必要な仮説 |

### 4.2 論拠の出典

主張の論拠は以下から導くこと:

- creation-space: 30 領域横断調査の結果
- awareness-space: 意識と気づきの構造分析
- kesson-driven-thinking: D1-D4 定義の正本

論拠なき主張は記載しない。調査が不十分な場合は「未解決」として明記する。
