# スライドコンテンツ生成ルール

> ドラフト（Markdown）の執筆時に従うルール。
> 本ファイルは変換層の正本であり、スキル（skills/rich-slides/）から参照される。

## 1. 語彙ルール

### 1.1 内部記号の禁止

以下の記号・用語をスライド内で使ってはならない:

| 禁止 | 代替表現 |
|---|---|
| D1, D2, D3, D4 | 概念を日常語で説明する（下記 §2 参照） |
| M1, M2, M3, M4 | 「意識の4層」「創造の5段階」等の平易な表現 |
| F軸 / O軸 | 「生存の軸」「信頼の軸」 |
| L0, L1, L2, L3 | 層番号を使わず機能で説明する |
| kesson-driven-thinking | 「欠損駆動思考」（日本語）または概念説明で代替 |

### 1.2 概念の言い換え辞書

| 内部概念 | 読者向けの表現 |
|---|---|
| D1（態度） | 棄却される誤差を、問いとして拾う態度 / 違和感を手放さない構え |
| D2（欠損） | 予測と現実のズレを「欠け」として感じる主観的経験 / 違和感・もやもや |
| D3（抱持） | 反射的に処理せず、問いとして保持すること / 引っかかりを手放さずにいる |
| D4（情動構成） | 生存に関わるか、関係に関わるかで、感じ方の方向が分かれる |
| Container | 保持を支える関係性の基盤 |

### 1.3 禁止フレーミング

| 禁止 | 理由 | 代替 |
|---|---|---|
| 「5ステップで〜」 | 自己啓発調 | 「5段階の構造」 |
| 「あなたの創造性を高める」 | 処方的 | 「今どこにいるか」 |
| 「本モデルが既存理論を統合する」 | 過大主張 | 「異なるラベルが同じ構造を指しているかを見る」 |
| 結論から始める | 憲章 §1 違反 | 問いから始める |

## 2. コンテンツ構造

### 2.1 メタスライドの3部構成

```
【Part 1: 問い】
  「無意味さは無意味なのか？」
  → 違和感を捨てずに問いに変える態度がある

【Part 2: 創造とは何か】
  30の学術領域が見つけた構造:
  → 創造は「欠けているもの」によって駆動される
  → 終わりは次の始まりの種になる（循環）
  → 境界は線ではなく、方向が選ばれる帯域
  → 詳細 → creation-space

【Part 3: 意識とは何か】
  身体 → ズレの検出 → 生存か信頼かの評価 → 問いとして保持
  → 意識は反応ではなく、問いを保持する能力
  → 詳細 → awareness-space

【帰結】
  欠損は欠落ではない。創造の駆動力である。
```

### 2.2 1スライド1メッセージ

- 各スライドは1つのメッセージだけを伝える
- 補足は箇条書き3項目以内
- 表は5行以内

### 2.3 外部参照の書き方

- 「30の学術領域を横断した調査によると...」→ creation-space リンク
- 「意識がズレを『欠け』として感じるとき...」→ awareness-space リンク
- リンクはスライド末尾に控えめに配置
- コンテンツの複製禁止（リンクのみ）

## 3. 文体基準

### 3.1 基本トーン: 一般的な解説書の文体

creation-space の解説文を基準とする。学術的・分析的な「である」調で、一般的な解説書として自然な文体を用いる。

| 基準 | 内容 |
|---|---|
| 文体 | 「である」調。3人称記述 |
| 主張 | dev/ 以下のリポジトリ調査結果から導かれたものに限定する |
| 論拠 | 主張には必ず出典または調査結果への参照を付与する |
| 未確定事項 | 確信度を明示する（§3.3 参照） |
| 方法論 | 必要に応じて方法論的原則を明記する |

### 3.2 禁止する文体

| 分類 | 具体例 | 理由 |
|---|---|---|
| 読者への語りかけ | 「どうしていますか？」「考えてみてください」 | 解説書ではない |
| 口語表現 | 「たぶん」「なんとなく」「だが」「〜の話だ」 | 文語で記述する |
| 会話体の列挙 | 「ひとつは…もうひとつは…」 | 表形式または箇条書きを使用する |
| 感情的修辞 | 「ざわつき」「もやもや」「ひとりでは待てない」 | 概念を定義で説明する |
| LLM チャット的創作 | エビデンスなき心理描写、架空の体験記述 | 調査結果に基づかない記述は禁止 |
| 自己啓発調 | 「5ステップで〜」「あなたの創造性を高める」 | 処方的表現は使わない |
| 過大主張 | 「統合した」「証明した」 | 構造的類似の指摘に留める |

### 3.3 確信度の表現

| レベル | 表現 | 例 |
|---|---|---|
| 確定 | 「〜である」 | 30領域中28領域で確認された |
| 有力 | 「〜と考えられる」 | 循環構造を示唆している |
| 仮説 | 「〜の可能性がある」 | 信頼が閾値を下げる触媒として機能する可能性 |
| 未解決 | 「〜は未解決である」 | 解体のメカニズムは記述できていない |

### 3.4 指月的態度

異なる領域の知見を「統合した」と言わない。
「異なるラベルが同じ構造を指しているかを見る」として記述する。

### 3.5 用語統一

| 禁止用語 | 正式用語 | 理由 |
|---|---|---|
| Withhold | 抱持 | 全文書で統一。Withhold は使用しない |

## 4. 論拠の要件

### 4.1 主張と論拠の対応

すべての主張は以下のいずれかに分類し、論拠を明示する:

| 分類 | 記法 | 説明 |
|---|---|---|
| [P] | 確立された事実 | 学術領域で広く認められた知見 |
| [M] | 構造的類似 | 異なる領域間の構造的対応の指摘 |
| [S] | 検証可能な仮説 | 今後の検証が必要な仮説 |

### 4.2 論拠の出典

主張の論拠は以下から導くこと:

- creation-space: 30 領域横断調査の結果
- awareness-space: 意識と気づきの構造分析
- kesson-driven-thinking: D1-D4 定義の正本

論拠なき主張は記載しない。調査が不十分な場合は「未解決」として明記する。

## 5. レビューチェックリスト

スライド生成後、以下を確認する:

- [ ] 内部記号（D1-D4, M1-M4, F軸/O軸, L0-L3）が含まれていない
- [ ] 問いから始まっている（結論先行でない）
- [ ] 1スライド1メッセージが守られている
- [ ] 文体が「である」調の解説書トーンである（§3.1-3.2 準拠）
- [ ] 口語表現・読者への語りかけ・LLM的創作がない（§3.2 準拠）
- [ ] すべての主張に論拠がある（§4 準拠）
- [ ] 外部参照はリンクのみ（コンテンツ複製なし）
- [ ] 確信度が適切に表現されている（§3.3 準拠）
- [ ] 「統合した」「証明した」等の過大主張がない
- [ ] 「Withhold」が使用されていない（「抱持」に統一）
