<!-- CHANGED(2026-03-25) 適用範囲をガイドにも拡張 -->

> ドラフト（Markdown）の執筆時に従うルール。
> 本ファイルはスライドおよび一般向けガイド（.md ドキュメント）の正本であり、スキル（transform/rich-slides/）から参照される。
> ガイド固有の差異は §3.4 で定義する。それ以外のルールはスライド・ガイド共通で適用する。

## 1. 語彙ルール

### 1.1 内部記号の禁止

以下の記号・用語をスライド内で使ってはならない:

| 禁止 | 代替表現 |
|---|---|
| D1, D2, D3, D4 | 概念を日常語で説明する（下記 §1.2 参照） |
| M1, M2, M3, M4 | 「意識の4層」「創造の5段階」等の平易な表現 |
| 生存軸 / 信頼軸 | 「生存の軸」「信頼の軸」 |
| L0, L1, L2, L3 | 層番号を使わず機能で説明する |
| kesson-driven-thinking | 「欠損駆動思考」（日本語）または概念説明で代替 |

### 1.2 概念の言い換え辞書

| 内部概念 | 読者向けの表現 |
|---|---|
| D1（態度） | 違和感を手放さない構え / 「おかしい」と感じたことを大切にする姿勢 |
| D2（欠損） | 予想と現実のズレを「何かが足りない」と感じること / 引っかかり |
| D3（抱持） | すぐに答えを出さず、問いとして持ち続けること |
| D4（情動構成） | 生存に関わるか、人との関係に関わるかで、感じ方の方向が分かれること |
| Container | 問いを持ち続けることを支えてくれる関係性 |

### 1.3 禁止フレーミング
<!-- CHANGED(2026-03-25) §3 文体基準に整合: 内部参照「憲章 §1 違反」を §3.2 準拠に変更 -->

| 結論から始める | §3.2 禁止事項 | 問いから始める |

## 2. コンテンツ構造

### 2.1 メタスライドの3部構成
<!-- CHANGED(2026-03-25) §3 文体基準に整合: リポジトリ名を排除 -->

```
【Part 1: 問い】
  「無意味さは無意味なのか？」
  → 違和感を捨てずに問いに変える態度がある

【Part 2: 創造とは何か】
  30の学術領域が見つけた構造:
  → 創造は「欠けているもの」によって駆動される
  → 終わりは次の始まりの種になる（循環）
  → 境界は線ではなく、方向が選ばれる帯域

【Part 3: 意識とは何か】
  身体 → ズレの検出 → 生存か信頼かの評価 → 問いとして保持
  → 意識は反応ではなく、問いを保持する能力

【帰結】
  欠損は欠落ではない。創造の駆動力である。
```

### 2.2 1スライド1メッセージ

- 各スライドは1つのメッセージだけを伝える
- 補足は箇条書き3項目以内
- 表は5行以内

### 2.3 外部参照の書き方
<!-- CHANGED(2026-03-25) §3 文体基準に整合: リポジトリ名の使用を禁止に変更 -->

- スライド内にリンク（URL）を埋め込むことは禁止
- リポジトリ名（creation-space、awareness-space 等）の使用は禁止。内容だけを説明する
- コンテンツの複製禁止

## 3. 文体基準

### 3.1 基本トーン: プレゼンテーション資料

<!-- CHANGED(2026-03-25) meta-overview.md をトーン見本として明示参照 -->

本スライドおよびガイドは公開向けプレゼンテーション資料である。Apple・Google の製品紹介ページ、企業の IR 説明資料、TED トークのスライドと同等の文体基準を適用する。

**文体見本**: `content/guides/meta-overview.md` を一般向けガイドのトーン基準とする。新規ガイド作成時は、このファイルの文体・語彙レベル・構成を参照し、同等のトーンで執筆すること。

| 基準 | 内容 |
|---|---|
| 文体 | 「です・ます」調。丁寧で平易な標準語 |
| 語彙 | 中学生が読んで意味がわかる日本語のみ使用する |
| 文の長さ | 1文40字以内を目安とする。短く、リズムよく |
| 構成 | 1スライド1メッセージ。補足は箇条書き3項目以内 |
| 前提知識 | ゼロ。この概念を初めて知る人が対象 |
| 視点 | 読者が「自分に関係ある」と感じられる具体性を持たせる |
| 情報密度 | 余白を活かす。詰め込まない。表は3行以内を推奨 |

### 3.4 ガイド固有のルール

<!-- CHANGED(2026-03-25) スライドとガイドの差異を定義 -->

本ルールファイルはスライドとガイドの両方に適用される。以下はガイド（.md ドキュメント）固有の差異である。

| 項目 | スライド | ガイド |
|---|---|---|
| 1文の長さ | 40字以内 | 60字以内（やや長い文も許容） |
| 文体基準（§3.1, §3.2） | そのまま適用 | そのまま適用（差異なし） |
| 禁止用語（§3.3） | そのまま適用 | そのまま適用（差異なし） |
| 構成 | 1スライド1メッセージ | 見出し → 短い導入（1-2文） → 箇条書きまたは短い段落 |
| 情報密度 | 表は3行以内推奨 | 表は5行以内許容 |
| 前提知識 | ゼロ | ゼロ（同じ） |

ガイドの構成パターン:

```
## 見出し（問いまたはテーマ）

導入文（1-2文で読者に文脈を伝える）

- 要点1
- 要点2
- 要点3

補足が必要な場合は短い段落（3文以内）で記述する。
```

## 4. 論拠の要件

### 4.1 主張と論拠の対応

すべての主張は以下のいずれかに分類し、論拠を明示する:

| 分類 | 記法 | 説明 |
|---|---|---|
| [P] | 確立された事実 | 学術領域で広く認められた知見 |
| [M] | 構造的類似 | 異なる領域間の構造的対応の指摘 |
| [S] | 検証可能な仮説 | 今後の検証が必要な仮説 |

| awareness-space | 意識と気づきの構造分析 | `/Users/uminomae/dev/awareness-space` |

## 6. ガイド生成ワークフロー

<\!-- CHANGED(2026-03-25) ガイド再生成時の手順を明文化 -->

ガイド（`kesson-general.md` 等）を新規作成または再生成するときは、以下の手順に従う。

### 6.1 事前準備

1. `content/guides/meta-overview.md` を読み、トーン・語彙レベル・構成を把握する
2. 既存のガイドがある場合は、内容の正確性を確認するため現行版を読む
3. 必要に応じて別リポジトリの知見を §4.2 の参照手順に従って取得する

### 6.2 執筆

1. §3.1（基本トーン）と §3.4（ガイド固有ルール）に従って執筆する
2. 冒頭は問いまたは日常の場面から始める。定義から入らない
3. 各セクションは1テーマに絞り、見出しで内容が予測できるようにする
4. 1文60字以内を守る。長い文は分割する
5. §3.3（禁止用語一覧）を随時参照し、違反がないか確認する
6. frontmatter に `generator_model` と `generated`（日付）を含める

### 6.3 自己検証

1. §5.1（共通チェック）の全項目を確認する
2. §5.3（ガイド固有チェック）の全項目を確認する
3. `content/guides/meta-overview.md` と並べて読み、トーンの乖離がないか確認する
4. 違反が見つかった場合は修正してから完了とする
