# エージェント運用ルール

## 標準エージェント（常時使用）

| 役割 | 機能 |
|------|------|
| プロセスガード | 品質ルール・プロジェクト定義との整合性を監視 |
| セッションヘルス | コンテキスト負荷・品質劣化を監視（CLAUDE.md §3 SH ガード） |

## 対話形式

- エージェント間で議論を進め、オーナーには選択肢と質問を提示
- オーナーのメタ認知を支援。一方的に答えを出さず、合意を明示確認

## エージェント指示の中立性原則

エージェントへの指示は、結果を誘導しない中立的な表現で記述する。
忖度（sycophancy）により、エージェントは指示に含まれる期待に合わせて結果を歪める傾向がある。

### 原則

| 誘導的（避ける） | 中立的（推奨） |
|-----------|----------|
| 「バグを探せ」 | 「ロジックを辿り、所見を報告せよ」 |
| 「この設計の問題点を指摘しろ」 | 「この設計を分析し、動作と構造を報告せよ」 |
| 「改善案を出せ」 | 「現状を記述し、選択肢があれば列挙せよ」 |
| 「正しいか検証しろ」 | 「主張とエビデンスの対応関係を報告せよ」 |

### 適用場面

- Claude Code / 外部エージェント（Gemini MCP 等）への指示書作成時
- 外部レビュー依頼の質問設計時
- 品質チェック指示時

## Gemini MCP 委譲パターン

kesson-space では、シェーダー/GLSL/Three.js メッシュ・マテリアルのタスクを Gemini MCP に委譲する（CLAUDE.md §6 委譲マトリクス参照）。

委譲時の注意:
- ユーザー許可を得てから委譲する
- 指示は中立的な表現で記述する（上記原則に従う）
- 結果は diff のみ抽出する（SH-5: 全文保持しない）

## 敵対エージェントパターン（3層品質保証）

忖度を逆手に取り、3つの役割を異なるセッション/エージェントに割り当てる品質保証パターン。

| 層 | 役割 | 忖度の利用 |
|----|------|-----------|
| 発見者 | 網羅的に問題・論点を洗い出す | 「見つけたい」方向に忖度させる |
| 反証者 | 発見者の出力を攻撃的に反証する | 「否定したい」方向に忖度させる |
| 審判 | 両者の出力を中立的にスコアリング | 「正確でありたい」方向に忖度させる |

### 適用ガイドライン

- 高信頼が必要なレビュー・検証時に使用
- 各層は別セッション（別コンテキスト）で実行すること
- 審判の出力はオーナーが最終確認する
- 全案件に適用する必要はない。高リスク・高重要度の判断時に使用
