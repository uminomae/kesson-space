# メモリと状態管理の棲み分けポリシー

## 2つの永続化レイヤー

| レイヤー | 場所 | スコープ | 用途 |
|---------|------|---------|------|
| **CLI メモリ** | `~/.claude/projects/{path}/memory/` | 会話をまたぐ | ユーザー意図・設計哲学・フィードバック・運用判断 |
| **リポジトリキャッシュ** | `.cache/` | セッション〜タスク単位 | 作業状態・タスクキュー・引き継ぎ |

## CLI メモリに書くもの

- オーナーのフィードバック・修正指示（type: feedback）
- プロジェクトレベルの意思決定・方針変更（type: project）
- 外部リソースへのポインタ（type: reference）
- オーナーの作業スタイル・好み（type: user）

**書かないもの**: コード構造、ファイルパス、Git 履歴、セッション状態、タスク進捗。
これらは CLAUDE.md・.cache/・git log から取得できる。

## .cache/ に書くもの

| ファイル | 内容 | 揮発性 |
|---------|------|--------|
| `session/state.md` | 現セッションの作業状態、HEAD SHA | セッション終了時に更新 |
| `backlog.md` | タスクキュー（優先度・依存関係） | 永続（タスク完了時に更新） |
| `session/handoff-*.md` | DT App <-> CLI 間の引き継ぎ | 読了後にアーカイブ |
| `session/log-*.md` | セッションログ | 永続（読み返し用） |
| `active/TASK-*.md` | 実行中タスクの指示書 | タスク完了時にアーカイブ |
| `inbox/` | 投入待ちタスク・レビュー結果 | 処理後にアーカイブ |

## 重複禁止の原則

CLI メモリと .cache/ に同じ情報を書かない。判断基準:
- **「次の会話で、.cache/ を読めば分かるか？」-> YES なら .cache/ だけに書く**
- **「.cache/ にはないが、次の会話で知っておくべきか？」-> YES なら CLI メモリに書く**
